# RespondAI — 게임 진행 명세서

## 핵심 컨셉

사용자와 AI가 번갈아 멜로디를 주고받는 **즉흥 연주 세션**. 승패 없음, HP 없음. 사용자가 얼마나 음악적으로 잘 호응했는지를 점수로 평가한다.

---

## 전체 게임 흐름

```
앱 시작
  → S1: 타이틀 화면 (모드 선택)
  → S2: 라운드 시작 화면 (조성 + BPM 고지)
  → S3: 메인 게임 화면 (입력 ↔ AI응답 × 3회)
  → S4: 라운드 결과 화면
  → [R1~R5 반복]
  → S5: 최종 결과 화면
```

---

## 라운드 설정

- 총 5라운드
- 매 라운드 시작 시 조성(Key)과 BPM을 랜덤 배정
- 조성 후보: C major, G major, D major, F major, Bb major, A minor, E minor, D minor, G minor, B minor
- BPM 후보: 70, 80, 90, 100, 110

### 라운드별 특수 조건 (UI에 텍스트로 표시)

| 라운드 | 화면 표시 문구 | 특이사항 |
|---|---|---|
| R1 | "자유롭게 연주하세요" | 없음 |
| R2 | "AI의 멜로디에 호응해보세요" | 없음 |
| R3 | "R1의 멜로디를 기억하나요?" | 모티프 계승 권장 |
| R4 | "리듬에 집중해보세요" | 없음 |
| R5 | "처음 멜로디로 마무리해보세요" | 모티프 활용 시 보너스 점수 |

---

## 1라운드 내 흐름

- 1라운드 = 유저와 AI가 각각 3번씩 주고받음 (총 6번)
- 항상 유저가 먼저, AI가 나중
- 순서: 유저→AI→유저→AI→유저→AI
- 이 3쌍을 **교환**이라고 부름 (교환 1, 교환 2, 교환 3)
- 3교환이 끝나면 라운드 종료 → 결과 화면

> 추후 교환 횟수를 3→5로 늘릴 수 있음. 상수 하나로 관리할 것.

---

## 노트 입력 규칙

- 유저는 가상 피아노 건반 클릭 또는 키보드 단축키로 노트를 입력
- 노트는 하나씩 순서대로 쌓임
- 최대 16노트까지 입력 가능. 16개가 되면 추가 입력 불가
- 최소 입력 제한 없음. 0개인 상태로 확정해도 됨
- 마지막으로 입력한 노트는 취소(백스페이스) 가능
- 입력 확정은 엔터 또는 [확정] 버튼
- 확정하면 입력한 노트가 피아노롤에 파란색으로 표시됨
- 확정 후에는 수정 불가

### 키보드 단축키

| 키 | 동작 |
|---|---|
| a s d f g h j k | 흰 건반 (C D E F G A B C) |
| w e t y u | 검은 건반 (C# D# F# G# A#) |
| Shift+↑ / Shift+↓ | 옥타브 올리기 / 내리기 |
| Backspace | 마지막 노트 취소 |
| Enter | 입력 확정 |

## AI 응답 규칙

- 현재는 **Mock 모드** 고정. 실제 모델 연동 전까지 변경하지 않음
- Mock 동작: 유저가 입력한 노트를 옥타브 하나 올려서 그대로 반환
  - pitch가 127을 넘는 경우 127로 고정
  - 유저가 빈 입력(0노트)을 확정하면 AI도 빈 응답 반환
- AI 응답이 도착하면 피아노롤에 빨간색으로 표시
- 유저 노트와 AI 노트 오디오를 순서대로 자동 재생 (유저 먼저, AI 나중)
- 오디오 재생이 끝나면 다음 교환으로 자동 진행

> 추후 A가 모델을 완성하면 Mock → Model로 전환. 이때 인터페이스는 동일하게 유지.

## Phase (단계) 관리

게임의 모든 UI 활성/비활성은 phase로 제어한다.

| phase | 의미 | 건반 활성 | 확정 버튼 |
|---|---|---|---|
| `user_input` | 유저 입력 대기 중 | ✅ | ✅ |
| `ai_response` | AI 응답 생성 + 재생 중 | ❌ | ❌ |
| `round_result` | 라운드 결과 화면 | ❌ | ❌ |
| `game_over` | 최종 결과 화면 | ❌ | ❌ |

- `ai_response` 중에는 건반 클릭, 엔터 입력 모두 무시
- `ai_response`가 끝나면 자동으로 `user_input`으로 전환 (3교환 완료 시 `round_result`로)

## 점수 계산 규칙

### 교환별 점수 (최대 1000점)

점수는 **유저가 입력한 노트**를 기준으로 계산한다.

**① 조성 일관성 (최대 300점)**
- 유저가 입력한 음들 중 현재 라운드 Key 스케일에 속하는 음의 비율 × 300
- 예: D minor 라운드에서 10음 중 8음이 D minor 스케일 → 240점
- 빈 입력이면 0점

**② 리듬 유사도 (최대 300점)**
- 직전 AI 응답과 유저 입력의 duration 패턴 유사도 × 300
- 교환 1(AI 응답이 없는 첫 입력)은 300점 만점 자동 부여
- 빈 입력이면 0점

**③ 모티프 활용 (최대 300점)**
- R1 교환 1에서 유저가 입력한 노트를 게임 전체의 **기준 모티프**로 저장
- 이후 입력에서 기준 모티프의 핵심 음형(첫 3~4음)이 등장하는 비율 × 300
- R1 교환 1은 300점 만점 자동 부여
- 빈 입력이면 0점

**④ 창의성 보너스 (최대 100점)**
- 직전 AI 응답과 완전히 동일하지 않으면 +50점
- 음정이나 리듬에 변형이 있으면 추가 +50점

**⑤ R5 모티프 보너스 (최대 150점)**
- R5에서 기준 모티프 활용도가 50% 이상이면 +150점

### 라운드 점수

- 라운드 점수 = 3번의 교환 점수 평균

### 라운드 등급

| 등급 | 점수 범위 |
|---|---|
| ⭐⭐⭐ PERFECT | 950점 이상 |
| ⭐⭐ GREAT | 800점 이상 |
| ⭐ CLEAR | 600점 이상 |
| TRY AGAIN | 600점 미만 (넘어가긴 함, 감점 없음) |

### 최종 점수 및 등급

- 최종 점수 = 5라운드 점수 합산 (최대 5000점)

| 등급 | 점수 범위 |
|---|---|
| S | 4500점 이상 |
| A | 3500점 이상 |
| B | 2500점 이상 |
| C | 2500점 미만 |

## 화면별 구성

### S1 — 타이틀

- 게임 제목 표시
- [🎹 피아노 모드 시작] [🎤 허밍 모드 시작 (Beta)] 버튼
- 버튼 클릭 시 모드 설정 후 S2로 이동

### S2 — 라운드 시작

- "Round X / 5" 표시
- 이번 라운드 Key + BPM 표시
- 라운드 특수 조건 문구 표시
- [시작] 버튼 → S3으로 이동

### S3 — 메인 게임

- 상단: Round X/5, Key, BPM, 현재 교환 X/3 표시
- 중앙: 피아노롤 (파란색=유저, 빨간색=AI, 교환이 쌓일수록 오른쪽으로 확장)
- 하단: 가상 피아노 건반
- 건반 아래: 현재 입력 중인 노트 목록 텍스트 표시
- 버튼: [← 취소] [▶ 미리듣기] [✅ 확정]
- phase가 `user_input`일 때 "당신의 차례" 표시
- phase가 `ai_response`일 때 "AI 응답 중..." 표시, 건반 및 버튼 비활성화

### S4 — 라운드 결과

- "Round X 결과" 표시
- 라운드 등급 (⭐ 개수 또는 TRY AGAIN)
- 라운드 점수 (XXX / 1000)
- 항목별 점수 바: 조성 / 리듬 / 모티프 / 보너스
- 이번 라운드 피아노롤 전체 요약
- R5이면 [결과 보기], 아니면 [다음 라운드] 버튼
- [다음 라운드] 클릭 시: 교환 관련 상태 초기화, Key/BPM 새로 배정, S2로 이동
- **r1_motif(기준 모티프)는 라운드가 넘어가도 초기화하지 않음**

### S5 — 최종 결과

- 최종 등급 (S/A/B/C) 애니메이션
- 총점 (XXXX / 5000)
- 라운드별 등급 요약 (R1 ⭐⭐ | R2 ⭐ | R3 ⭐⭐⭐ ...)
- 전체 세션 피아노롤 (5라운드 전체)
- [다시하기] 버튼 → S1으로, 전체 상태 초기화

## 데이터 보존 규칙

| 데이터 | 초기화 시점 |
|---|---|
| r1_motif (기준 모티프) | 게임 전체에서 유지. [다시하기] 시에만 초기화 |
| exchange_log | 라운드 종료 시 초기화 |
| current_notes | 교환 확정 시 초기화 |
| round_history | [다시하기] 시에만 초기화 |

## 상수 목록

클로드 코드는 아래 값들을 상수로 분리해서 관리할 것. 나중에 조정이 필요할 수 있음.

| 상수명 | 값 | 설명 |
|---|---|---|
| MAX_EXCHANGES | 3 | 라운드당 교환 횟수 (추후 5로 변경 가능) |
| MAX_NOTES | 16 | 교환당 최대 입력 노트 수 |
| DEFAULT_DURATION | 2 | 기본 노트 길이 (16분음표 단위) |
| TOTAL_ROUNDS | 5 | 전체 라운드 수 |
| AI_MODE | "mock" | "mock" 또는 "model" |
