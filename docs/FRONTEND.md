## 프론트엔드 결정

**Gradio (`gr.Blocks()`) 단일 앱**으로 확정.

기존 Streamlit은 클릭마다 전체 페이지가 리렌더링되는 구조로, 가상 피아노 건반 표시·실시간 피아노롤 업데이트·게임 상태 관리에 구조적으로 부적합하여 교체한다. Gradio는 Python 백엔드(FluidSynth, music21, PESTO, PyTorch 모델)와 동일 런타임에서 동작하므로 subprocess 없이 직접 연동 가능하며, 발표 데모 환경(로컬/Colab)에서 `python app.py` 한 줄로 실행된다.

| 컴포넌트 | 용도 |
|---|---|
| `gr.Blocks()` | 전체 레이아웃 자유 구성 |
| `gr.HTML()` | 가상 피아노 건반 (인라인 CSS+JS, 키보드 단축키 포함) |
| `gr.Plot()` | 피아노롤 matplotlib 렌더링 |
| `gr.HTML()` | HUD, 분석 미터, 원형 에너지 시각화 SVG 실시간 렌더링 |
| `gr.Audio()` | FluidSynth 렌더링 오디오 재생 (`autoplay=True`) |
| `gr.Audio(sources=["microphone"])` | 허밍 모드 마이크 입력 |
| `gr.State()` | 게임 상태 관리 |

## 화면 구성 (5개 화면)

| 코드 | 화면 이름 | 설명 |
|---|---|---|
| S1 | 타이틀 / 모드 선택 | 게임 제목, 피아노/허밍 모드 선택 |
| S2 | 라운드 시작 | 랜덤 조성(Key) + BPM 제시 |
| S3 | 메인 게임 화면 | 입력 + 피아노롤 + 원형 시각화 + HUD |
| S4 | 라운드 결과 | 점수, 피드백, 다음 라운드 진행 |
| S5 | 최종 결과 | 등급, 전체 히스토리 피아노롤, 다시하기 |

## 게임 상태 관리

`gr.State()`로 아래 게임 상태를 유지한다:

| 키 | 설명 |
|---|---|
| `round` | 현재 라운드 (1~5) |
| `exchange` | 현재 라운드 내 교환 번호 (1~3) |
| `score` | 누적 총점 |
| `combo` | 연속 고득점 횟수 |
| `current_notes` | 현재 입력 중인 노트 목록 |
| `ai_notes` | 최신 AI 응답 노트 |
| `round_results` | 라운드별 점수 결과 목록 |
| `r1_motif` | R1 첫 교환 노트 (게임 전체 기준 모티프) |
| `phase` | 현재 단계 (`user_input` / `ai_response` / `round_result` / `game_over`) |
| `key` | 현재 라운드 조성 |
| `bpm` | 현재 라운드 BPM |
| `mode` | 입력 모드 (`piano` / `humming`) |

## 핵심 UI 요구사항

### 1. 가상 피아노 건반

- `gr.HTML()`로 흰 건반 + 검은 건반이 정확히 렌더링되는 피아노 건반 구현
- HTML은 완전한 인라인 CSS+JS로 작성 (외부 CDN 의존성 최소화, 필요 시 cdnjs.cloudflare.com만 허용)
- 건반 클릭 시 해당 MIDI pitch 번호가 Python 백엔드로 전달됨
- 클릭된 건반은 시각적으로 눌린 상태로 표시 (색상 변화)
- 마우스가 없는 환경을 고려해 **키보드 단축키로도 입력 가능**

### 2. 실시간 피아노롤 시각화

- 사용자 입력 노트: **파란색**
- AI 응답 노트: **빨간색**
- matplotlib 또는 plotly로 그린 피아노롤을 `gr.Plot()`으로 표시
- 노트 입력될 때마다 피아노롤 즉시 업데이트

### 3. 게임 HUD

- 현재 라운드 표시 (Round X/5)
- 현재 교환 표시 (교환 X/3)
- 현재 조성(Key) + BPM
- 누적 점수 + 콤보

### 4. 분석 미터 (라운드 결과 후 표시)

- Key match / Rhythm / Motif 3개 게이지 (`gr.HTML` SVG로 구현)

### 5. 버튼

| 버튼 | 동작 |
|---|---|
| [입력 완료] | 현재까지 입력한 노트를 확정하고 AI 응답 요청 |
| [재생] | 입력한 멜로디 오디오 재생 |
| [AI 응답 듣기] | AI 생성 멜로디 오디오 재생 |
| [다음 라운드] | 다음 라운드 진행 |
| [🎤 허밍 모드 Beta] | `gr.Audio(sources=["microphone"])`로 마이크 입력 |

### 6. 결과 화면 (S5)

- 최종 등급 (S/A/B/C) 애니메이션
- 전체 5라운드 히스토리 피아노롤

### 실시간 시각화 요소 (VIZ)

캐릭터 없음. 캐릭터 자리는 **원형 에너지 시각화**로 대체. SVG + Python 코드로 생성하며 별도 이미지 제작 불필요.

| 코드 | 이름 | 용도 | 데이터 소스 | 생성 방식 |
|---|---|---|---|---|
| VIZ-01 | 플레이어 원형 에너지 시각화 | S3 좌측 플레이어 영역 | 플레이어 입력 pitch → 각도, velocity → 반지름 | SVG + Python 코드 생성 |
| VIZ-02 | AI 원형 에너지 시각화 | S3 우측 AI 영역 | 모델 attention weight → 색상·반지름 | SVG + Python 코드 생성 (A 연동 필요) |

**색상 매핑 규칙:**

| 시각화 | 색조(Hue) | 채도(Saturation) | 밝기(Lightness) |
|---|---|---|---|
| VIZ-01 플레이어 | pitch 낮음=파랑(240°) → 높음=보라(270°) | velocity 값 → 0~100% | 고정 50% |
| VIZ-02 AI | attention 낮음=주황(40°) → 높음=빨강(0°) | 고정 70% | 생성 진행률 → 30~70% |
